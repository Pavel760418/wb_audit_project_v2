# Аудит кабинета Wildberries (версия 2)

Модуль полного аудита кабинета продавца Wildberries (стандартная подписка, без WB Джем).
Собирает Excel-отчёты в единый BI-файл: продажи, остатки, финансы, юнит-экономика,
риски и план действий. Работает по принципу «мягкой деградации» (graceful degradation) —
отсутствие файла не останавливает расчёт, а помечает соответствующий блок как частичный
или нерассчитанный, без подстановки вымышленных значений.

## Возможности

- Загрузка отчётов Wildberries и ручных справочников через веб-интерфейс на русском языке.
- Мультизагрузка недельных финансовых отчётов: несколько недель объединяются в единый период,
  дубликаты строк удаляются автоматически, различия в названиях колонок нормализуются.
- Готовые Excel-шаблоны для ручного заполнения (себестоимость, доп. реклама, классификатор, корректировки).
- Устойчивость к типовым ошибкам загрузки (не тот файл, пустой файл, лишние строки над шапкой,
  устаревший формат .xls, разный набор колонок в недельных отчётах).
- Итоговый Excel-дашборд со статусами полноты данных и блоком ограничений анализа.

## Структура проекта

```
wb_audit_project_v2/
├── config.py                # реестр отчётов, группы, карта колонок, названия блоков
├── main.py                  # CLI-вход (в т.ч. для PyCharm) + единая pipeline-функция
├── streamlit_app.py         # веб-интерфейс загрузки, расчёта и скачивания
├── modules/
│   ├── loader.py            # устойчивое чтение Excel и «мягкая деградация»
│   ├── finance_merge.py     # объединение нескольких недельных финансовых отчётов
│   ├── templates.py         # генерация Excel-шаблонов для ручного заполнения
│   ├── data_map.py          # карта доступности данных, статусы блоков, ограничения
│   ├── analytics.py         # расчёты продаж, финансов, юнит-экономики, рисков
│   ├── excel_builder.py     # примитивы Excel: статусы, светофор, таблицы
│   └── report_builder.py    # сборка итогового Excel-файла
├── requirements.txt
├── .gitignore
└── README.md
```

## Установка и запуск в PyCharm

1. Откройте папку проекта в PyCharm как новый проект.
2. Убедитесь, что выбран интерпретатор виртуального окружения проекта
   (File → Settings → Python Interpreter).
3. Установите зависимости:
   ```
   python -m pip install -r requirements.txt
   ```
4. Запустите `main.py` — модуль пошагово запросит пути к отчётам и период анализа.
   Для еженедельного финансового отчёта можно указать несколько файлов через запятую.
5. Итоговый файл появится в папке `output/`.

## Запуск веб-версии локально

```
streamlit run streamlit_app.py
```

Интерфейс разбит на понятные разделы: обязательные данные, дополнительные данные,
ручные справочники, период анализа, статус загруженных файлов, запуск расчёта,
скачивание итогового Excel и скачивание шаблонов.

## Несколько недельных финансовых отчётов

Финансовые отчёты в кабинете Wildberries формируются понедельно. Чтобы проанализировать
месяц, квартал или произвольный длинный период, загрузите сразу несколько недельных файлов:

- 4 недельных файла ≈ один месяц;
- 8–12 файлов ≈ длинный период.

Модуль объединит их в единый датафрейм, удалит полностью дублирующиеся строки,
нормализует немного отличающиеся названия колонок и продолжит расчёт по тем файлам,
которые удалось прочитать. В статусах видно, сколько файлов загружено, сколько вошло
в расчёт, какие обработаны с предупреждениями и какие не вошли.

## Excel-шаблоны для ручного заполнения

Прямо в приложении (раздел «Скачать шаблоны») можно скачать готовые шаблоны:

- «Себестоимость SKU»;
- «Дополнительные рекламные расходы»;
- «Классификатор SKU / Категория / Бренд»;
- «Ручные корректировки».

Каждый шаблон содержит лист с данными (жирная закреплённая шапка, читаемая ширина колонок,
одна строка-пример) и лист «Инструкция». Не переименовывайте колонки — модуль ориентируется
на их названия.

## Публикация на GitHub и Streamlit Community Cloud

1. Создайте новый репозиторий на GitHub.
2. Загрузите содержимое папки (без виртуального окружения, `input/`, `output/` — они в `.gitignore`).
3. На https://share.streamlit.io нажмите **Create app**.
4. Выберите репозиторий, ветку и файл **streamlit_app.py** как основной.
5. Нажмите **Deploy** и получите ссылку на приложение.

## Обязательные и опциональные отчёты

| Отчёт | Обязательность |
|---|---|
| Еженедельный финансовый отчёт (детализация) | Обязателен для базового анализа (можно несколько недель) |
| Себестоимость SKU (ручная таблица) | Нужна для полной юнит-экономики SKU |
| Отчёт «Продажи» | Желателен для расширенной аналитики трендов |
| Остатки на складах / Оборачиваемость | Желательны для блока рисков дефицита |
| Платное хранение / Реклама | Желательны для точной финансовой модели |

Если отчёт не загружен — соответствующий лист итогового Excel получит статус
«Частичный расчёт» или «Нет данных для расчёта», без фиктивных значений.
