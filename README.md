# WB Audit Project (v2)

Модуль полного аудита кабинета продавца Wildberries (стандартная подписка, без WB Джем).
Собирает Excel-отчёты в единый BI-файл с продажами, остатками, финансами, юнит-экономикой,
рисками и планом действий. Работает по принципу graceful degradation — отсутствие файла
не останавливает расчёт, а просто помечает соответствующий блок как частичный/нерассчитанный.

## Структура проекта

```
wb_audit_project_v2/
├── config.py              # реестр отчётов, маппинг колонок, минимально достаточные наборы данных
├── main.py                 # CLI-вход для PyCharm (пошаговый запрос файлов и периода)
├── streamlit_app.py        # веб-оболочка для тестовой загрузки и отладки ingestion-слоя
├── modules/
│   ├── loader.py            # загрузка Excel с graceful degradation
│   ├── data_map.py          # карта доступности данных и статусов блоков
│   ├── analytics.py         # расчёты продаж, финансов, юнит-экономики, рисков
│   ├── excel_builder.py     # примитивы Excel: светофор, статусы, таблицы
│   └── report_builder.py    # сборка итогового Excel-файла
├── requirements.txt
├── .gitignore
└── README.md
```

## Установка и запуск в PyCharm

1. Откройте папку проекта в PyCharm как новый проект.
2. Убедитесь, что выбран интерпретатор `.venv` этого проекта (File → Settings → Python Interpreter).
3. Откройте Terminal внизу PyCharm и выполните:
   ```
   python -m pip install -r requirements.txt
   ```
4. Запустите `main.py` — модуль пошагово спросит пути к отчётам и период анализа.
5. Итоговый файл появится в папке `output/`.

## Запуск веб-версии локально (для теста перед GitHub/Streamlit Cloud)

```
streamlit run streamlit_app.py
```

## Публикация на GitHub и Streamlit Community Cloud

1. Создайте новый репозиторий на GitHub (например `wb-audit-v2`).
2. Загрузите туда содержимое этой папки (без `.venv`, `input/`, `output/` — они уже в `.gitignore`).
3. Зайдите на https://share.streamlit.io, нажмите **Create app**.
4. Выберите репозиторий, ветку `main` и файл **streamlit_app.py** как основной.
5. Нажмите **Deploy** и получите ссылку на приложение.

## Обязательные и опциональные отчёты

| Отчёт | Обязательность |
|---|---|
| Еженедельный финансовый отчёт (детализация) | Обязателен для базового анализа |
| Ручная таблица себестоимости | Обязательна для полной юнит-экономики SKU |
| Отчёт «Продажи» | Желателен для расширенной аналитики трендов |
| Остатки на складах / Оборачиваемость | Желательны для блока рисков дефицита |
| Платное хранение / Реклама | Желательны для точной финансовой модели |

Если отчёт не загружен — соответствующий лист итогового Excel получит статус
«Частичный расчёт» или «Нет данных для расчёта», без фиктивных значений.
